# Studio Training Path

Onboarding → Branding → Growth

---

## What This Is

A learning sequence for studios adopting Tandava. It exists to:

- Reduce operational risk
- Prevent premature customization
- Align learning with studio maturity

This path follows how studios actually evolve, not how software vendors wish they did.

**Companion resource:** [Checklists](CHECKLISTS.md) — Printable self-assessment tools for each phase

---

## Core Principles

1. **Reality First** — Training follows the realities of running a studio, not the feature set of the platform.

2. **Readiness Over Completion** — Studios move between phases based on capability and confidence, not checklists.

3. **Re-Entry Is Expected** — Studios revisit earlier phases as their business changes. This is maturity, not regression.

---

# Phase 1: Onboarding

**Goal: Operational Trust**

### The Question You're Answering

> "Can I run my studio tomorrow without things breaking?"

If the answer is not "yes", nothing else matters.

### What You're Optimizing For

- Stability
- Predictability
- Familiarity
- Confidence that you chose correctly

You are *not* yet optimizing for differentiation, aesthetics, or growth.

### What to Focus On

| Task | Why It Matters |
|------|----------------|
| Create your studio | Foundation of everything |
| Add locations | Where classes happen |
| Define offerings | What you teach (class types) |
| Schedule sessions | When classes happen |
| Accept bookings | Members can sign up |
| View rosters | Know who's coming |
| Assign staff roles | Who does what |

This phase makes the software **disappear into the background**.

### What to Skip (For Now)

| Skip This | Why |
|-----------|-----|
| Visual branding | Adds complexity before stability |
| Page customization | Creates fragile setups |
| Advanced integrations | Premature optimization |
| Analytics deep-dives | Data needs time to accumulate |
| Growth tactics | Amplifies instability |

### Success Condition

> "I stopped thinking about the software."

If you're still actively thinking about the system, onboarding isn't complete.

# Phase 2: Branding

**Goal: Ownership and Confidence**

### The Question You're Answering

> "How do I make this feel like *my* studio, without breaking what works?"

Branding is about emotional ownership, not decoration.

### Why Branding Comes Second

Branding before stability feels risky.
Branding after stability feels empowering.

Once operations are trusted, customization becomes intentional instead of anxious.

### The Two Design Surfaces

This phase teaches the critical separation:

| Surface | What It Includes | Customization Level |
|---------|------------------|---------------------|
| **Internal (Staff)** | Scheduling, rosters, admin | Light (logo, colors only) |
| **Consumer (Public)** | Website, booking, events | Full (your brand shines here) |

**Why this matters:**
- Staff tools stay fast and familiar
- Customer pages express your identity
- Neither interferes with the other

### What to Focus On

| Task | Result |
|------|--------|
| Apply brand colors | Everything adapts automatically |
| Upload logo | Appears across both surfaces |
| Choose fonts | Typography follows your brand |
| Customize consumer pages | Your website, your way |
| Review before publishing | Catch issues early |

### What to Skip (For Now)

| Skip This | Why |
|-----------|-----|
| Redesigning admin layouts | Stability regression |
| Changing internal density | Staff will struggle |
| "Making admin look cool" | Wrong surface for expression |
| Complex page structures | Keep it maintainable |

### Success Condition

> "Clients say: 'This feels like us.'
> Staff say: 'Nothing feels harder.'"

If either side feels friction, branding went too far.

# Phase 3: Growth

**Goal: Operational Leverage**

### The Question You're Answering

> "How do we do more without working harder?"

Growth here means reducing effort per outcome, not rapid expansion.

### Why Growth Comes Last

Growth amplifies existing systems.

| If your system is... | Growth will... |
|----------------------|----------------|
| Disorganized | Multiply chaos |
| Inconsistent | Multiply confusion |
| Stable and owned | Multiply effectiveness |

Only stable, owned systems should scale.

### What to Focus On

| Task | Benefit |
|------|---------|
| Use reports to inform decisions | Data-driven choices |
| Export data for accounting | Clean financial records |
| Add integrations intentionally | Extend without complexity |
| Support multiple locations | Grow your footprint |
| Delegate to staff confidently | Reduce your workload |
| Recognize when *not* to add | Protect simplicity |

### What to Skip (Always)

| Skip This | Why |
|-----------|-----|
| Hustle culture | Burnout isn't growth |
| Vanity metrics | Measure what matters |
| Feature hoarding | Unused features are debt |
| Premature automation | Manual is fine until it isn't |

### Success Condition

> "We're doing more, and it feels easier."

If it feels harder, you entered this phase prematurely.

## Re-Entry Is Expected

Studios commonly revisit phases:

| Situation | Re-enter |
|-----------|----------|
| Opening a new location | Phase 1 (Onboarding) |
| Launching a new program | Phase 2 (Branding) |
| Staff turnover | Phase 1 or 2 |
| Seasonal reset | Any phase as needed |

This is maturity, not failure. The path is cyclical.

---

## Studio Size Adaptations

| Studio Type | Path Behavior |
|-------------|---------------|
| **Tiny (1-2 instructors)** | Onboarding is light; Branding may happen quickly; Growth may never be needed |
| **Multi-location** | Longer onboarding; Formal branding phase; Growth becomes necessary |
| **Hybrid (classes + trainings + events)** | Branding repeats per program; Growth focuses on delegation |

The structure stays the same. The pace adapts.

---

## Philosophy

This path aligns:
- Capability with choice
- Confidence with customization
- Growth with readiness

Stability comes first.
Ownership comes next.
Leverage comes last.
